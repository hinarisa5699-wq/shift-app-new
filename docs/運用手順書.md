## 0. Pythonのインストール（初回のみ）

このアプリを動かすには「Python（パイソン）」というソフトウェアが必要です。
パソコンに入っていない場合は、以下の手順でインストールしてください。

> **すでにPythonがインストール済みかわからない場合は、先に確認する方法があります（後述の「インストールの確認方法」を参照）。**

### インストール手順

1. ブラウザ（Chrome や Edge）を開きます
2. 画面上部のアドレスバー（URLが表示される場所）に `https://www.python.org` と入力して Enter キーを押します
3. 画面上部のメニューにある「**Downloads**」にマウスを合わせます
4. 黄色い「**Download Python 3.xx.x**」と書かれたボタンをクリックします（数字はその時点の最新版です）
5. ファイルのダウンロードが始まります。画面下部または「ダウンロード」フォルダに `python-3.xx.x-amd64.exe` というファイルが保存されます
6. ダウンロードしたファイルをダブルクリックして開きます
7. インストール画面が表示されます。**ここが一番重要です**:

> **必ず画面の一番下にある「Add python.exe to PATH」にチェック（レ点）を入れてください。**
> ここにチェックを入れないと、アプリが正しく動きません。

8. 「**Install Now**」をクリックします
9. インストールが始まります。進行バーが右端まで行くのを待ちます（1〜3分程度）
10. 「Setup was successful」と表示されたら「**Close**」をクリックします

### インストールの確認方法

正しくインストールできたか確認します。

1. キーボードの「**Windows**」キー（左下のWindowsマークのキー）を押します
2. そのまま `cmd` とキーボードで入力します
3. 「**コマンド プロンプト**」と表示されるので、クリックするか Enter キーを押します
4. 黒い画面が開いたら、以下の文字を入力して Enter キーを押します:

```
python --version
```

5. 「`Python 3.11.x`」や「`Python 3.12.x`」のようにバージョン番号が表示されれば成功です

> もし「'python' は、内部コマンドまたは外部コマンド〜として認識されていません」と表示された場合は、Pythonのインストール時に「Add python.exe to PATH」のチェックが入っていなかった可能性があります。Pythonをアンインストール（「設定」→「アプリ」→「Python」→「アンインストール」）してから、上記の手順でもう一度やり直してください。

## 1. アプリの起動と停止

### 初回セットアップ（最初の1回だけ行います）

1. 納品された ZIP ファイルを「**デスクトップ**」など、わかりやすい場所に保存します
2. ZIP ファイルを**右クリック**して、メニューから「**すべて展開...**」を選びます
3. 「展開先の選択」画面が出たら、そのまま「**展開**」をクリックします
4. 展開されたフォルダの中にある「**shift-app**」フォルダを開きます
5. フォルダの中にある「**setup_windows.bat**」というファイルをダブルクリックします

> 「WindowsによってPCが保護されました」という青い画面が表示された場合は、「**詳細情報**」をクリックし、次に「**実行**」をクリックしてください。これはセキュリティの警告であり、問題はありません。

6. 黒い画面（コマンドプロンプト）が開き、自動的にセットアップが進みます:

| 表示される文字 | 意味 |
|---|---|
| `[1/4] Python command check...` | Pythonが入っているか確認中 |
| `[2/4] Create virtual environment...` | アプリ専用の環境を作成中 |
| `[3/4] Install dependencies...` | 必要な部品をインストール中 |
| `[4/4] Setup complete.` | セットアップ完了 |

7. 「**Setup complete.**」と表示されたらセットアップ成功です
8. 「何かキーを押してください」と表示されたら、任意のキー（Enterキーなど）を押して画面を閉じます

> セットアップでエラーが出た場合は、Pythonがインストールされていない可能性があります。[セクション0](#0-pythonのインストール初回のみ) に戻って、Pythonのインストールを行ってください。

---

### 通常起動（毎回行う操作）

パソコンを起動した後や、アプリを使いたいときは、以下の手順で起動します。

1. デスクトップ（またはZIPを展開した場所）の「**shift-app**」フォルダを開きます
2. フォルダの中にある「**start_windows.bat**」をダブルクリックします
3. 黒い画面（コマンドプロンプト）が開き、以下のように表示されます:

```
Starting shift app...

========================================
  ブラウザで以下のURLを開いてください:
  http://localhost:5050
========================================
```

4. **ブラウザが自動的に開きます**。もし開かない場合は、ブラウザで `http://localhost:5050` にアクセスしてください
5. アプリの画面が表示されます

> **重要: 黒い画面（コマンドプロンプト）は絶対に閉じないでください。**
> 黒い画面を閉じるとアプリが停止し、ブラウザから操作できなくなります。
> 黒い画面は最小化（画面右上の「−」ボタン）しておけば邪魔になりません。

---

### 停止方法（アプリを終了するとき）

アプリの使用が終わったら、以下のどちらかの方法で停止できます。

**方法1: ×ボタンで閉じる（かんたん）**

1. 黒い画面（コマンドプロンプト）の右上にある「**×**」ボタンをクリックします
2. アプリが停止します

**方法2: キーボードで停止する**

1. 黒い画面（コマンドプロンプト）をクリックして選択します
2. キーボードの「**Ctrl**」キーを押しながら「**C**」キーを押します
3. アプリが停止します

> ブラウザの画面は閉じてもアプリは停止しません。アプリを完全に停止するには、必ず黒い画面を閉じてください。

## 2. 毎月のシフト作成手順

毎月、以下の流れでシフトを作成してください。
目安として、シフト作成の **1週間前** までに休み希望を集め、**月末** までに翌月分を作成するとスムーズです。

### ステップ1: 職員情報の確認・更新

新しい月のシフトを作る前に、まず職員の情報が最新かどうか確認します。

1. アプリの画面で「**職員管理**」を開きます
2. 以下の内容を確認します:
   - 新しく入った職員がいれば登録する（→ 操作マニュアル「1. 職員を登録する」）
   - 退職・異動した職員がいれば削除または情報を修正する
   - 勤務条件に変更があれば編集する（勤務曜日の変更、時間帯の変更など）

### ステップ2: 休み希望の登録

1. 各職員から**翌月の休み希望**を集めます（紙のメモやLINEなどで）
2. アプリで職員ごとに休み希望日を登録します（→ 操作マニュアル「2. 休み希望を登録する」）
3. **前月の休み希望**が残っている場合は削除します

> 休み希望の登録を忘れると、その職員は「希望なし」として扱われ、どの日にも配置される可能性があります。

### ステップ3: 条件設定の確認

1. アプリの画面で「**条件設定**」を開きます
2. 最低配置人数などの設定が正しいか確認します
3. 変更がなければそのままでOKです

### ステップ4: シフト生成

1. アプリの画面で「**シフトカレンダー**」を開きます
2. 翌月の**年月を選択**します（例: 2026年4月）
3. 「**シフト生成**」ボタンをクリックします
4. 生成中の画面になります。**約1〜2分**お待ちください

> 生成中は、ページを閉じたり他の操作を行ったりしないでください。

### ステップ5: 結果の確認

1. シフトが表示されたら、以下を確認します:
   - **赤い警告メッセージ**が出ていないか
   - 各職員の配置が適切か（偏りがないか）
   - 休み希望がきちんと反映されているか
   - 各日に必要な人数が配置されているか
2. 問題がある場合は、職員情報や条件設定を調整して、もう一度「シフト生成」をクリックします

### ステップ6: ダウンロードと印刷

1. 確認が済んだら「**Excel出力**」ボタンをクリックします
2. Excel ファイルがダウンロードされます
3. ダウンロードしたファイルを開いて、必要に応じて印刷します
4. 印刷したシフト表を職員に配布します

---

### 月次作業チェックリスト

毎月のシフト作成で漏れがないよう、以下のリストを活用してください。

| チェック | 作業内容 |
|:---:|---|
| &#9744; | 職員情報が最新か確認した |
| &#9744; | 前月の休み希望を削除した |
| &#9744; | 翌月の休み希望を全員分登録した |
| &#9744; | 条件設定を確認した |
| &#9744; | シフトを生成した |
| &#9744; | 警告がないことを確認した（またはやむを得ない警告を把握した） |
| &#9744; | Excel ファイルをダウンロードした |
| &#9744; | 印刷して職員に配布した |
| &#9744; | データのバックアップを取った（→ [セクション4](#4-データのバックアップ)） |

## 3. 職員の入退職時の対応

### 新しい職員が入った場合

1. アプリの画面で「**職員管理**」を開きます
2. 「**新規登録**」をクリックします
3. 以下の情報を入力します:

| 入力項目 | 説明 | 入力例 |
|---|---|---|
| 氏名 | 職員の名前 | 山田花子 |
| 部門 | ケア または 調理 | ケア |
| 雇用形態 | 常勤 または パート | パート |
| 保有資格 | 該当する資格にチェック | 介護福祉士 |
| 兼務可否 | 訪問（ハウス）にも入れるか | 可 |
| 勤務可能曜日 | 勤務できる曜日にチェック | 月・火・木・金 |
| 勤務時間帯 | 勤務できる時間帯 | 9:00〜18:00 |
| 連勤上限 | 連続して勤務できる最大日数 | 5日 |
| 週勤務日数上限 | 1週間の最大勤務日数 | 5日 |

4. 入力が終わったら「**登録**」をクリックします
5. 翌月以降のシフト生成に自動的に反映されます

### 職員が退職した場合

1. アプリの画面で「**職員管理**」を開きます
2. 退職する職員を探します
3. その職員の「**削除**」ボタンをクリックします
4. 確認のメッセージが表示されたら「**OK**」をクリックします

> **注意**: 職員を削除すると、その職員に関するデータ（休み希望など）もすべて削除されます。その職員が含まれる月のシフトをまだExcelに出力していない場合は、先にExcel出力してから削除してください。

### 勤務条件が変わった場合

以下のような場合に、職員情報を編集します:

- パートから常勤への変更
- 勤務曜日の追加・削減
- 兼務可否の変更
- 連勤上限の変更

**操作手順:**

1. アプリの画面で「**職員管理**」を開きます
2. 該当する職員の「**編集**」をクリックします
3. 変更したい項目を修正します
4. 「**保存**」をクリックします
5. 翌月のシフトを再生成すれば反映されます

## 4. データのバックアップ

アプリのデータ（職員情報、休み希望、シフトなど）はすべて「**shift.db**」という1つのファイルに保存されています。`shift.db` は `shift-app` フォルダの**1つ上の階層**に保存されます。このファイルをコピーしておけば、万が一のときにデータを復元できます。

### バックアップの頻度（目安）

| タイミング | 理由 |
|---|---|
| 毎月のシフト生成前 | 万が一のやり直しに備える |
| 職員を大量に登録・変更した後 | 入力ミス時にやり直せるようにする |
| 月に1回（月初など） | 定期的な保険として |

---

### バックアップ手順（ファイルのコピー）

1. アプリが動いている場合は**停止します**（[停止方法はこちら](#停止方法アプリを終了するとき)）
2. 「**shift-app**」フォルダがある場所を開きます
3. `shift-app` フォルダと**同じ階層**にある「**shift.db**」というファイルを見つけます

> 「shift.db」が見つからない場合は、ファイルの拡張子が表示されていない可能性があります。エクスプローラーの上部メニューで「**表示**」→「**ファイル名拡張子**」にチェックを入れてください。

4. 「**shift.db**」を**右クリック**して「**コピー**」を選びます
5. 同じ階層の何もない場所で**右クリック**して「**貼り付け**」を選びます
6. 「**shift - コピー.db**」というファイルが作られます
7. このファイルを**右クリック**して「**名前の変更**」を選びます
8. わかりやすい名前に変更します（例: `shift_バックアップ_2026年3月1日.db`）

> バックアップファイルは、USB メモリや別のフォルダにもコピーしておくとさらに安心です。

---

### 復元手順（バックアップからデータを戻す）

何らかの理由でデータを以前の状態に戻したい場合は、以下の手順で復元します。

1. アプリを**停止します**（[停止方法はこちら](#停止方法アプリを終了するとき)）
2. 「**shift-app**」フォルダがある場所を開きます
3. 現在の「**shift.db**」を**右クリック**して「**名前の変更**」を選びます
4. 名前を「**shift_旧データ.db**」などに変更します（削除せずに残しておくと安心です）
5. バックアップファイル（例: `shift_バックアップ_2026年3月1日.db`）を**右クリック**して「**コピー**」を選びます
6. 同じ階層の何もない場所で**右クリック**して「**貼り付け**」を選びます
7. 貼り付けたファイルを**右クリック**して「**名前の変更**」を選びます
8. 名前を「**shift.db**」に変更します

> ファイル名は必ず「shift.db」にしてください。名前が違うとアプリがデータを読み込めません。

9. 「**start_windows.bat**」をダブルクリックしてアプリを起動します
10. データが復元されていることを確認します

## 5. トラブルシューティング

困ったときは、以下の表からお困りの症状を探してください。

### 起動に関するトラブル

| 症状 | 原因と対処方法 |
|---|---|
| 「setup_windows.bat」をダブルクリックしても何も起きない | ファイルの種類が「.bat」ではなく「.txt」になっている可能性があります。ファイルを右クリック →「プロパティ」で種類を確認してください。また、セキュリティソフトがブロックしている場合もあります |
| 「`[ERROR] Python 3 is not installed.`」と表示される | Pythonがインストールされていません。[セクション0](#0-pythonのインストール初回のみ) の手順でPythonをインストールしてください |
| 「`[ERROR] venv is not initialized.`」と表示される | 初回セットアップがまだ完了していません。先に「setup_windows.bat」をダブルクリックして実行してください |
| 「`[ERROR] Failed to create virtual environment.`」と表示される | Pythonのインストールが不完全な可能性があります。Pythonをアンインストールして、もう一度インストールし直してください（「Add python.exe to PATH」のチェックを忘れずに） |
| 青い画面で「WindowsによってPCが保護されました」と表示される | 「**詳細情報**」→「**実行**」の順にクリックしてください。これはWindows標準のセキュリティ警告であり、問題ありません |
| 「`Address already in use`」と表示される | 前回のアプリがまだ動いています。黒い画面（コマンドプロンプト）がどこかに開いていないか確認し、すべて閉じてから「start_windows.bat」を再実行してください |

---

### ブラウザに関するトラブル

| 症状 | 原因と対処方法 |
|---|---|
| ブラウザに何も表示されない / 「このサイトにアクセスできません」と出る | 以下を確認してください: (1) アドレスバーの入力が `http://127.0.0.1:5050` で正しいか（`https` ではなく `http` です） (2) 黒い画面に「Running on http://127.0.0.1:5050」と表示されているか (3) 黒い画面が閉じていないか |
| カレンダーが表示されない | ブラウザの「再読み込み」ボタン（アドレスバー横の丸い矢印）をクリックしてみてください |
| 画面のレイアウトが崩れている | ブラウザを変えてみてください（**Google Chrome** の使用を推奨します）。または、キーボードの「Ctrl」を押しながら「Shift」を押しながら「Delete」を押して「閲覧データの削除」画面を開き、「キャッシュされた画像とファイル」にチェックを入れて「データを削除」をクリックしてください |

---

### シフト生成に関するトラブル

| 症状 | 原因と対処方法 |
|---|---|
| 「シフトを生成中...」のまま進まない | 生成には1〜2分かかります。しばらくお待ちください。5分以上経っても終わらない場合は、ブラウザの再読み込みボタンをクリックしてください |
| 生成後に赤い警告が大量に出る | 職員の数が、必要な配置人数に対して足りていない可能性があります。職員を追加するか、「条件設定」の最低人数を見直してください |
| 特定の日に全員が「休み」になっている | その日に勤務できる職員がいない状態です（全員が休み希望 or 固定休）。休み希望を調整してください |
| 前月のシフトが消えた | 同じ月のシフトを再生成すると上書きされます。Excel出力済みであれば問題ありません |

---

### データに関するトラブル

| 症状 | 原因と対処方法 |
|---|---|
| 間違えて職員を削除してしまった | バックアップがあれば [復元手順](#復元手順バックアップからデータを戻す) で戻せます。バックアップがない場合は、もう一度手動で登録してください |
| 「shift.db」を間違えて削除してしまった | バックアップファイルがあれば、名前を「shift.db」に変更してください。バックアップもない場合は、開発者にご連絡ください。初期状態に復元できます |
| アプリが動かなくなった | 黒い画面を閉じて「start_windows.bat」を再実行してください。それでも直らない場合は、パソコンを再起動してからもう一度試してください |
| パソコンを再起動した後にアプリが使えない | パソコンを再起動するとアプリも終了しています。「start_windows.bat」をダブルクリックするだけで、アプリを再び起動できます。セットアップ（setup_windows.bat）のやり直しは不要です |

## 6. 付録

### ファイル構成（参考）

通常の運用で中のファイルを直接触る必要はありませんが、参考として記載します。

```
配置先フォルダ/
├── shift.db              ← データベース（★バックアップ対象）
└── shift-app/
    ├── app.py                ← アプリ本体（変更しないでください）
    ├── setup_windows.bat     ← 初回セットアップ用（最初の1回だけ実行）
    ├── start_windows.bat     ← 通常起動用（毎回これをダブルクリック）
    ├── requirements.txt      ← 必要な部品の一覧（変更しないでください）
    ├── docs/
    │   ├── 操作マニュアル.md  ← 画面の操作方法
    │   └── 運用手順書.md      ← この手順書
    └── venv/                 ← アプリ専用の環境（変更しないでください）
```

> 普段使うファイルは「**start_windows.bat**」（起動）と「**shift.db**」（バックアップ）の2つだけです。

### お問い合わせ

アプリの不具合や追加のご要望がありましたら、**クラウドワークスのメッセージ機能**で開発者までご連絡ください。

その際、以下の情報をお伝えいただけるとスムーズに対応できます:

| お伝えいただきたい情報 | 例 |
|---|---|
| 困っている内容 | 「シフト生成ボタンを押しても動かない」 |
| いつから起きているか | 「昨日までは動いていた」 |
| 黒い画面に表示されているエラー | スクリーンショット（画面の写真）があると助かります |

> **スクリーンショットの撮り方**: キーボードの「**Windows**」キーを押しながら「**Shift**」キーを押しながら「**S**」キーを押すと、画面の一部を切り取って保存できます。
